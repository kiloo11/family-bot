# Семейный органайзер (Telegram-бот + сайт)

Категории: 🎁 Желания (с фото, ссылкой и историей выполненных/забронированных),
🎬 Фильмы (с отметкой «просмотрено» и оценкой
1-5 ⭐), 💰 Оплаты (с напоминаниями),
📷 Альбомы (несколько фото на альбом + дата + страна/город). Данные общие — что добавил в
боте, увидишь на сайте, и наоборот: оба сервиса читают один и тот же SQLite-файл через
общий Docker volume.

Навигация в боте — единая: постоянные кнопки под окном ввода сообщения, без дублей.

## Быстрый старт (бот)

1. Получи токен у [@BotFather](https://t.me/BotFather): `/newbot`.
2. Скопируй `.env.example` в `.env` и впиши `BOT_TOKEN`.
   - Если хочешь, чтобы напоминания об оплатах падали в общий семейный чат —
     добавь бота в группу, напиши там любое сообщение и через
     [@userinfobot](https://t.me/userinfobot) или `getUpdates` узнай `chat_id` группы
     (обычно отрицательное число), впиши его в `FAMILY_CHAT_ID`.
   - Если оставить `FAMILY_CHAT_ID` пустым — напоминания шлются лично тому, кто добавил платёж.
3. Запусти:

```bash
docker compose up -d --build
```

4. Проверь логи: `docker compose logs -f`
5. В Telegram напиши боту `/start`.

## Быстрый старт (сайт)

Сайт использует **Telegram Login (OIDC)** — актуальную (с 2026 года) схему входа через
Telegram: Authorization Code Flow + PKCE, со своим Client ID/Secret и проверкой подписанного
JWT-токена. Старый iframe-виджет с `/setdomain` Telegram отключил — теперь только эта схема.
Требование то же: **нужен HTTPS-домен**. На голом IP или `http://` войти не получится —
Telegram потребует зарегистрированный Redirect URI на HTTPS.

1. Сгенерируй секрет для сессий сайта:
   ```bash
   python3 -c "import secrets; print(secrets.token_hex(32))"
   ```
   Впиши его в `.env` как `WEB_SECRET_KEY`.

2. Заведи домен, указывающий на VPS (A-запись), и подними перед сайтом HTTPS-реверс-прокси
   *до* того, как идти в BotFather — Telegram проверяет доступность Redirect URI при
   добавлении. Проще всего — [Caddy](https://caddyserver.com/): сам получает сертификат
   Let's Encrypt.
   ```
   fam.твой-домен.ру {
       reverse_proxy localhost:8000
   }
   ```
   → `/etc/caddy/Caddyfile`, затем `sudo systemctl restart caddy`. Подойдёт и nginx + certbot.

3. В @BotFather: выбери бота → **Bot Settings → Web Login**. Там:
   - в **Redirect URIs** добавь `https://fam.твой-домен.ру/` (именно так, как открывается
     сайт — со слэшем на конце, если Telegram сам его добавил, важно точное совпадение)
   - скопируй **Client Secret** (Client ID тебе не нужен — впиши в `.env` только Client
     Secret, ID мы сами возьмём из `BOT_TOKEN`)

4. В `.env` впиши:
   ```
   TELEGRAM_CLIENT_SECRET=<то, что скопировал в BotFather>
   TELEGRAM_REDIRECT_URI=https://fam.твой-домен.ру/
   ```
   `TELEGRAM_REDIRECT_URI` должен **посимвольно** совпадать с тем, что зарегистрировано в
   BotFather — лишний/отсутствующий слэш на конце сломает вход.

5. (Опционально) Ограничь вход только своей семьёй: узнай свой telegram id через
   [@userinfobot](https://t.me/userinfobot) и впиши в `.env`:
   ```
   ALLOWED_TG_IDS=111222333,444555666
   ```
   Если оставить пустым — войти сможет кто угодно, кто откроет сайт и нажмёт кнопку логина
   (нормально, если ссылка на сайт не публикуется нигде, кроме семьи).

6. Запусти (если ещё не запускал):
   ```bash
   docker compose up -d --build
   ```
   Сайт слушает порт из `WEB_PORT` (по умолчанию 8000) — именно на него должен
   проксировать Caddy/nginx.

7. Открой `https://fam.твой-домен.ру` и нажми «Войти через Telegram».

## Где данные

SQLite-файл и загруженные с сайта фото лежат в `./data/` (проброшено через volume в оба
контейнера) — переживут пересборку и перезапуск. Просто бэкапь папку `data/`, если важно.

## Как это устроено

- **Бот**: aiogram 3, диалоги добавления через FSM (`app/states.py`)
- **Сайт**: FastAPI (`web/main.py`) отдаёт JSON API + собранный React SPA
  (`web/frontend`, Vite). `docker compose up --build` собирает фронтенд внутри
  `Dockerfile.web` (стадия `frontend-build`) и кладёт `dist/` в `web/static/dist` —
  руками ничего собирать не нужно. Для локальной разработки фронтенда отдельно от
  Docker: `cd web/frontend && npm install && npm run dev` (порт 5173, проксирует
  `/api`, `/auth`, `/logout`, `/media` на FastAPI на `:8000`, который надо поднять
  отдельно, например `uvicorn web.main:app --reload`)
- Бот и сайт **используют один и тот же код доступа к базе** (`app/db.py`) — оба Docker-образа
  копируют папку `app/`, но каждый ставит только нужные себе зависимости (сайту не нужен aiogram)
- **Авторизация на сайте**: Telegram Login (OIDC) — Authorization Code Flow с PKCE
  (`web/auth.py`). Мы редиректим на `oauth.telegram.org`, получаем обратно `code`,
  меняем его на подписанный JWT (`id_token`) через сервер-to-сервер запрос с Client
  Secret, затем проверяем подпись токена по публичным ключам Telegram (JWKS) — только
  после этого верим данным пользователя. Сессия сайта — отдельная подписанная cookie
  (`itsdangerous`), 30 дней
- **Фото**: бот хранит Telegram `file_id`; сайт для таких фото ходит в Bot API на лету
  (`GET /media/tg/<file_id>`). Фото, загруженные через сайт, физически лежат в
  `data/uploads/` и помечены в базе как `web:<имя_файла>` — бот их тоже умеет присылать
  (см. `app/photos.py`), просто открывает файл с диска вместо обращения к Telegram
- **APScheduler** — раз в день (час задаётся `REMINDER_HOUR`) проверяет, у каких платежей
  срок наступает в течение `REMINDER_DAYS_BEFORE` дней, и шлёт напоминание
- **Повторяющиеся платежи**: после «✅ Готово» бот/сайт не закрывают платёж, а переносят
  срок на следующий месяц
- **История желаний**: как только желание забронировали («🙋 Забронировать») или отметили
  выполненным («✅ Готово»), оно уходит из активного списка в «🗂 История» (в боте —
  кнопка в меню категории, на сайте — переключатель-вкладка сверху списка). В истории видно,
  кто забронировал или что оно выполнено, и можно нажать «↩️ Вернуть в активные», если
  бронь/покупка сорвалась
- **Фильмы**: «✅ Готово» переносит фильм из «Хотим посмотреть» в «Просмотрено» (в боте —
  кнопка `✅ Просмотрено` в меню категории, на сайте — переключатель-вкладка сверху списка).
  После этого можно поставить оценку 1-5 ⭐ — в боте сразу предлагается после отметки
  «готово» и повторно через «⭐ Оценить» в списке просмотренных, на сайте — кликом по звезде
  прямо на карточке
- **Альбомы**: фото хранятся в отдельной таблице `album_photos` — у альбома их может быть
  сколько угодно. Удаление альбома каскадно удаляет все его фото
- Все элементы (желания/фильмы/оплаты/альбомы) хранятся в одной таблице `items` с полем
  `category` — так проще добавлять новые разделы что в боте, что на сайте

## Как добавить новую категорию (например, «🛒 Покупки»)

**В боте:**
1. `app/keyboards.py` → добавить в `CATEGORY_TITLES`
2. `app/states.py` → новая `StatesGroup` с нужными полями
3. `app/handlers/add_item.py` → диалог по аналогии с `AddMovie`
4. `app/handlers/list_items.py` → добавить ветку в `format_item()`, если нужен свой формат карточки

**На сайте:**
1. `web/frontend/src/categories.js` → добавить категорию в объект `CATEGORIES` (заголовок,
   иконка, эндпоинт, поля формы) — вкладка в сайдбаре и форма добавления появятся сами
2. `web/main.py` → эндпоинт `POST /api/<категория>` по аналогии с `api_add_movie`
3. Если категории нужен переключатель вида «активные/история» — добавить её в `SUBTABS`
   там же, в `categories.js`
